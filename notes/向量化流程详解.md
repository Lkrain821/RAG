# 向量化（Embedding）完整流程详解

> 从一句话到一串数字，模型内部到底做了什么？

---

## 总览

```
"我喜欢猫"
  → 分词 → 查词表得初始向量
  → 经过 12 层 Transformer Block（每层含 Self-Attention + Feed-Forward）
  → 取最后一层所有词向量做平均（池化）
  → 得到 512 维句向量
```

---

## 第一步：分词（Tokenization）

```
"我喜欢猫"
    │
    ▼
["我", "喜欢", "猫"]
    │  查词汇表（模型内置字典）
    ▼
[1057, 3821, 892]
```

每个 Token 对应一个数字 ID，模型只认数字不认字。

---

## 第二步：查词嵌入矩阵，得初始向量

```
Token ID         查表（词嵌入矩阵，训练好的）
────────         ──────────────────────────
1057 ("我")   → x₁ = [ 0.10, -0.20,  0.30,  0.05 ]
3821 ("喜欢") → x₂ = [ 0.40,  0.15, -0.25,  0.08 ]
892  ("猫")   → x₃ = [-0.30,  0.22,  0.12,  0.03 ]
```

> **注意**：示例用 4 维便于演示，真实模型用 512 维。此时三个向量各不相关。

---

## 第三步：经过 12 层 Transformer Block

每一层的结构完全相同：

```
输入向量
    │
    ├──→ Multi-Head Self-Attention ──→ Add & Norm ──┐
    │                                                │
    └──→ Feed-Forward Network ──────→ Add & Norm ──┴──→ 输出（送下一层）
```

### 3.1 Multi-Head Self-Attention（多头自注意力）

#### 3.1.1 生成 Q、K、V 向量

每个词的初始向量 × 三个权重矩阵 = Q、K、V：

```
x₁ × W_Q = Q₁      x₁ × W_K = K₁      x₁ × W_V = V₁
x₂ × W_Q = Q₂      x₂ × W_K = K₂      x₂ × W_V = V₂
x₃ × W_Q = Q₃      x₃ × W_K = K₃      x₃ × W_V = V₃
```

| 钥匙 | 全称 | 含义 | 类比 |
|------|------|------|------|
| Q | Query | "我在找什么？" | 求职者的技能需求 |
| K | Key | "我有什么？" | 简历上的标签 |
| V | Value | "我的干货？" | 面试的实际内容 |

**W_Q、W_K、W_V 是模型训练出来的矩阵，初始为随机数，通过反向传播不断调整。**

#### 3.1.2 Q × K 计算注意力分数

用每个词的 Q 去"问"所有词的 K，得到相关性分数：

```
Q₁·K₁  Q₁·K₂  Q₁·K₃      [0.021,  0.018,  0.011]
Q₂·K₁  Q₂·K₂  Q₂·K₃  =   [0.019, -0.002,  0.008]
Q₃·K₁  Q₃·K₂  Q₃·K₃      [0.007, -0.016,  0.025]
```

分数越大 = 两个词越相关。

#### 3.1.3 Softmax 归一化

对每一行做 Softmax，变成 0~1 的权重（每行总和为 1）：

```
                           "我"     "喜欢"   "猫"
原始分数                  Softmax 后
[0.021, 0.018, 0.011]    [0.338,  0.337,  0.325]   ← "我"关注所有词较均匀
[0.019,-0.002, 0.008]    [0.339,  0.332,  0.329]   ← "喜欢"关注所有词较均匀
[0.007,-0.016, 0.025]    [0.334,  0.326,  0.340]   ← "猫"稍微更关注自己
```

#### 3.1.4 权重 × V = 融合后的新向量

用权重矩阵对 V 向量做加权求和：

```
"猫"的新向量 = 0.334×V₁ + 0.326×V₂ + 0.340×V₃

V₁ = [ 0.06, -0.01,  0.16,  0.00] × 0.334 → [ 0.020,-0.003, 0.053, 0.000]
V₂ = [ 0.11,  0.05, -0.11,  0.05] × 0.326 → [ 0.036, 0.016,-0.036, 0.016]
V₃ = [-0.13,  0.03,  0.06,  0.01] × 0.340 → [-0.044, 0.010, 0.020, 0.003]
                                              ─────────────────────────────
                                            = [ 0.012, 0.023, 0.037, 0.019]
```

#### 3.1.5 Multi-Head：并行执行 8 次

上面只是 **Head 1**。同时还有 Head 2~8，每组有自己的 W_Q、W_K、W_V：

```
Head 1（语法结构）：    W_Q₁  W_K₁  W_V₁  → 输出 4 维
Head 2（情感色彩）：    W_Q₂  W_K₂  W_V₂  → 输出 4 维
Head 3（实体关系）：    W_Q₃  W_K₃  W_V₃  → 输出 4 维
...
Head 8（共现模式）：    W_Q₈  W_K₈  W_V₈  → 输出 4 维
```

8 个头的结果拼接：

```
[a,b,c,d, e,f,g,h, ..., y,z,.,.]  ← 8 × 4 = 32 维
```

#### 3.1.6 × W_O 融合多头信息

```
32 维拼接向量 × W_O(32×4) = 4 维输出

W_O 和 W_Q/W_K/W_V 一样，都是训练出的权重矩阵
作用：把 8 个不同视角的发现融合成一个紧凑表示
```

**Self-Attention 输出完成。**

### 3.2 残差连接 + LayerNorm（Add & Norm）

```
output = LayerNorm(输入 + Attention输出)
```

**残差连接（Add）**：把原始输入加回来，防止深层网络退化。

```
类比：小组讨论
  - 你发言前有自己的观点（输入）
  - 听了别人发言后调整了观点（Attention 输出）
  - 最终观点 = 原始观点 + 调整量
  - 你不会完全抛弃原始观点
```

**LayerNorm**：把数值拉回标准范围，防止越跑越偏。

### 3.3 Feed-Forward Network（前馈网络）

```
FFN(x) = ReLU(x × W₁ + b₁) × W₂ + b₂

这是一个两层的全连接网络，每个词独立执行
作用：增加非线性变换能力，让模型能学到更复杂的模式
```

### 3.4 再次 Add & Norm

```
output = LayerNorm(Step3.2输出 + FFN输出)
```

### 3.5 完整一层的数据流

```
输入: x₁, x₂, x₃
  │
  ├── Multi-Head Self-Attention ──→ a₁, a₂, a₃
  │    ├─ Head 1: QK attention → V 加权 → 4维
  │    ├─ Head 2: QK attention → V 加权 → 4维
  │    ├─ ...
  │    └─ Head 8: QK attention → V 加权 → 4维
  │         ↓ 拼接 32维
  │         ↓ × W_O(32×4)
  │         → 4维
  │
  ├── Add & Norm: a' = LayerNorm(x + a)
  │
  ├── Feed-Forward: f = ReLU(a'×W₁+b₁)×W₂+b₂
  │
  ├── Add & Norm: output = LayerNorm(a' + f)
  │
  └── 输出 = 下一层的输入
```

重复 12 次。

---

## 第四步：池化（Pooling）→ 句向量

第 12 层输出三个词的终极向量后，取平均：

```
"我"   → [0.014, 0.016, 0.028, 0.013]
"喜欢" → [0.013, 0.018,-0.009, 0.016]
"猫"   → [0.015, 0.020, 0.022, 0.015]
         ───────────────────────────── 对应位置相加 ÷ 3
       = [0.014, 0.018, 0.014, 0.015]  ← 4 维句向量
```

真实模型输出 512 维：

```
[0.13, -0.32, 0.57, 0.02, -0.41, ...]  ← 512 个浮点数
```

## 关键问题解答

### 为什么是 512 维？

- 维度是模型设计时定的，BGE-small 的隐藏层宽度就是 512
- 这是**效果与成本的权衡**：维度越高越精准，但越慢越占空间
- 不同模型维度不同：BGE-base 768，BGE-large 1024，OpenAI 1536
- 512 是 2 的幂次，GPU 计算效率最高

### W_Q、W_K、W_V、W_O 从哪来？

- 训练初期全是**随机数**
- 通过海量文本训练，用**反向传播 + 梯度下降**自动调出来的
- 没有人手工填写任何一个数字
- 训练目标是让语义相近的句子最终向量距离也近

### 残差连接为什么要加？

- 深层网络容易退化（层数越多效果反而变差）
- 残差连接保证"最差也能保持原样"，信息不会在深层丢失
- 类比：你听了别人意见后，最终观点 = 原始观点 + 调整量，而非完全替换
