=============================================================================
一、学习路线与核心概念
=============================================================================

文章来源：牛客网 腾讯后端实习生"永泽one"的 AI 应用开发学习路线

核心观点：
- AI 应用工程师 = 传统后端 + 大模型接入
- Agent 四大组件：大脑(LLM)、手脚(Tools/MCP)、记忆(Memory)、规划(Planning)
- 两阶段学习：AI 扫盲 + AI 应用基础（Prompt Engineering / RAG）
- RAG 是 Agent 技术栈中最核心、商业化最广泛的技术

[1] 幻觉 (Hallucination)
- 定义：模型生成看似合理但完全虚构的内容
- 根本原因：LLM 本质是"预测下一个 Token"而非"查找事实"
- 三种类型：事实性幻觉、忠实性幻觉、逻辑性幻觉
- 解决方案：RAG（最有效）、降低 Temperature、允许说不知道、要求引用来源

[2] Prompt Engineering
- 结构化三板斧：Role（角色）+ Task（任务）+ Constraint（约束）
- 输出控制：用 JSON 而非自然语言，便于程序解析
- Few-shot：给示例比纯描述更有效
- 六个检查点：角色、任务、格式、长度、边界、示例

[3] Pydantic + LLM 的局限性
- Pydantic 只校验格式（类型正确），不校验内容（语义正确）
- 三层方案：Prompt 引导 → Pydantic 校验格式 → RAG 锚定事实

[4] Transformer 和 QKV
- Q（Query）："我在找什么？"
- K（Key）："我这里有什么？"
- V（Value）："我的实际内容是什么？"

[5] 其他基础概念
- Token：模型最小处理单元，中文 1 字 ≈ 1-2 Token
- 上下文窗口：GPT-4o 128K, Claude 4 200K, DeepSeek-V3 128K
- Temperature：控制创造力，0=保守，>1=发散
- Embedding：文本 → 向量，语义相近=向量距离近

二、RAG 系统搭建
=============================================================================

架构：
  离线阶段：文档 → 文本分割 → 向量化(Embedding) → 向量数据库
  在线阶段：用户提问 → 向量化 → 检索相关块 → 拼接Prompt → LLM生成答案

[doc_loader.py] 文档加载与文本分割
  - TextLoader 加载 txt 文件
  - RecursiveCharacterTextSplitter 分割文本
  - 参数：chunk_size=500, chunk_overlap=50
  - 创建 test.txt 测试文档（RAG 介绍文本）
  - 结果：1 篇文档 → 2 个文本块

[build_index.py] 向量化与存储
  - Embedding 模型：BAAI/bge-small-zh-v1.5（智源开源中文模型，本地运行）
  - 向量数据库：ChromaDB（轻量级，本地存储）
  - 存储目录：./chroma_db/
  - 搜索测试："什么是 RAG？" 成功检索到最相关文档块

[chat.py] 完整问答（交互式）
  - LLM：Ollama + qwen3:4b（本地运行，2.5GB）
  - 流程：提问 → 检索(k=3) → 拼接上下文 → 构建Prompt → LLM生成答案
  - Prompt 策略：角色+约束+只基于文档+不知道就说不知道
  - 运行方式：
      venv\Scripts\activate
      python chat.py
  - 输入 quit / exit / q 退出，Ctrl+C 安全退出

问答效果（问"RAG 有什么优点？"）：
  1. 解决大模型的知识过时问题
  2. 让大模型能够访问私有数据
  3. 有效减少幻觉

五、遇到的问题与解决
=============================================================================

[问题 1] langchain-community 包废弃警告
  原因：LangChain 将社区集成拆分成独立包
  解决：安装 langchain-huggingface / langchain-chroma / langchain-ollama
        更新 import 路径，Ollama 改为 OllamaLLM

[问题 2] Ollama 未启动时报错 ConnectionRefusedError (localhost:11434)
  原因：chat.py 中 llm.invoke() 时才检测连接
  解决：增加 try/except，捕获连接错误并给出启动 Ollama 的友好提示

[问题 3] HuggingFace Hub 未认证警告
  原因：未设置 HF_TOKEN
  解决：在 venv/Scripts/activate.bat 中添加 HF_TOKEN 环境变量

[问题 4] 终端中文乱码
  原因：Windows 终端 GBK 编码 vs Python UTF-8 输出
  状态：部分终端正常，未完全解决

[问题 5] WebFetch 调用失败（首次抓取文章时）
  原因：WebFetch 是 deferred tool，不知道完整参数列表，漏传 prompt
  解决：先用 ToolSearch 加载工具 schema，再调用

[问题 6] input() 在部分终端触发 KeyboardInterrupt
  原因：stdin 异常关闭或终端兼容性问题
  解决：用 try/except 捕获 KeyboardInterrupt 和 EOFError
